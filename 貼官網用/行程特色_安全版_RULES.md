## 正常基線（已鎖定標籤）：「回到上一版正常版」＝下面這一版

用 **Git 標籤（tag）** 在遠端記錄「官網貼上後、每日行程／地圖仍正常」的檔案快照。日後可 **整支檔還原到該標籤**。

**目前狀態：尚未鎖定第一版基準。** 待官網貼上並確認無誤後，請：

1. `git commit` + `git push` 當前 `行程特色_貼上版.html`／`行程特色_開發版.html`（及本次相關文件）。
2. 建立標籤，建議命名：**`pe55-feature-baseline-YYYYMMDDhhmm`**（例：`pe55-feature-baseline-202604091200`）。
3. `git push origin pe55-feature-baseline-YYYYMMDDhhmm`（或 `git push origin --tags`）。
4. **更新本文件下表**與 **`專案最新備註.md`**、**`.cursor/rules/peru-55-continue.mdc`** 內的預設標籤名稱。

### 口頭／協作約定（含 Cursor 助理）

當有人說 **「改回上一版正常」「還原正常版」「回到可貼版」** 時：

1. **先確認**是否要還原到本文件鎖定的**目前正常基線標籤**所指向之兩支 HTML。  
2. 若要更舊的 tag 或某次 commit，**問清再** `git checkout`。  
3. 預設語意：**「上一版正常」＝本節表格中的目前基準標籤**。

**還原兩支行程特色檔（不動專案其他檔案）** — 將 `<標籤>` 換成實際名稱：

```bash
git fetch origin
git checkout <標籤> -- 貼官網用/行程特色_貼上版.html 貼官網用/行程特色_開發版.html
```

## 最重要的一條（與夏古號同一後台之實測）

> **此 CMS 的 `<style>` 標籤數量上限為 8 個。**  
> 超過則後方官方區塊（每日行程、地圖、團費說明等）可能**全部消失**。  
> **實務請維持 ≤7**；`行程特色_貼上版.html` 應將 CSS **合併為單一** `<style type="text/css">`。若需 Google Fonts，**僅能**在該包**最開頭**使用 **一條** `@import`，且勿讓 `<style>` 總數逼近上限。

> 即使只有 1 個 `<style>`，整段 HTML **過長**仍可能遭後台**截斷**或變慢。異常時先懷疑長度與巢狀複雜度，並洽廠商確認欄位上限。

### 關於 `<style>`／`<script>`／`<link>`

- `<style>`：**總數 ≤8（建議 ≤7）**；貼上版建議 **1 個**。
- `<script>`：**建議 1 個連續區塊**（與夏古號貼上版策略相同）。
- `<link>`／`<meta>`／`<title>`／`<base>`：原則避免塞在內文；必要時先問廠商。

## 每次上線前固定流程

1. 只使用 `貼官網用/行程特色_貼上版.html` 的內容。
2. 全選複製，貼到後台「行程特色」欄位。
3. 發佈後檢查：行程特色區、**每日行程**、**地圖**、頁尾是否正常。
4. 若異常，還原至上一基準標籤或上一 commit。

## 新增內容的方法（依安全程度）

### 方法 A：加圖片（最安全）

```html
<!-- [SCREEN-XX START] -->
<div style="width:100vw;margin-left:calc(50% - 50vw);line-height:0;">
  <img src="圖片網址" alt="描述" style="width:100%;display:block;" loading="lazy" />
</div>
<!-- [SCREEN-XX END] -->
```

### 方法 B：需要新 CSS（謹慎）

- 新規則寫進**既有唯一** `<style>` 底部（註解分段），**勿**新增第二個 `<style>` 除非已確認總數仍安全。

## 禁用與命名

- 不可使用易與官網主題衝突的 id（如 `id="hero..."`）。
- 本專案新增之 id／class 請一律加前綴 **`pe55-`**（與夏古號之 `tc-` 區隔，避免日後混貼混淆）。
- 根容器：**`#pe55-cms-safe-root`**；內容區建議 **`#pe55-root`**。
- **互動狀態**：class 只加在 **`#pe55-cms-safe-root` 內**，**禁止**對 `document.documentElement`／`<html>` 加 class。
- 動效請顧及 **`prefers-reduced-motion`**。

## 快速除錯

1. 刪最新區塊 → 重貼 → 再測。  
2. 數 `<style>` 是否超過 8。  
3. 數 `<script>`；貼上版宜為 1。  
4. `git checkout` 還原至基準標籤。
